# Neroes — Polar H10 Pipeline

Physiological data analysis pipeline for Polar H10 heart-rate sessions exported from [flow.polar.com](https://flow.polar.com).

---

## Project layout

```
neroes_polar_pipeline/
├── data/
│   ├── raw/           ← place participant CSVs here (see naming convention)
│   ├── processed/     ← parquet files generated by the notebooks
│   └── reports/       ← auto-generated HTML EDA report
├── notebooks/
│   ├── 01_data_ingestion.ipynb
│   ├── 02_cleaning_and_engineering.ipynb
│   └── 03_eda_and_analysis.ipynb
├── src/               ← reusable Python modules
├── config/
│   └── participants.yaml
└── requirements.txt
```

---

## How to add a new participant

1. **Register them** in `config/participants.yaml`:
   ```yaml
   - id: "P003"
     name: "Maria"
     folder: "P003_Maria"
     age: 30
     sex: "F"
     height_cm: 168
     weight_kg: 65
     hr_max: null
     hr_rest: 62
     vo2max: null
   ```

2. **Create their data folder:**
   ```
   data/raw/P003_Maria/
   ```

3. **Drop the exported CSVs** following the naming convention below.

---

## CSV naming convention

```
data/raw/<ID_Name>/<Name>_YYYY-MM-DD.csv
```

| Part | Example | Rule |
|------|---------|------|
| ID_Name | `P003_Maria` | Must match the `folder` field in `participants.yaml` |
| Name | `Maria` | Human-readable label |
| YYYY-MM-DD | `2026-06-01` | ISO date of the session |

Full example: `data/raw/P003_Maria/Maria_2026-06-01.csv`

---

## How to export CSVs from Polar Flow

1. Log in at [flow.polar.com](https://flow.polar.com) → Training → select session.
2. Click the **export** icon → **Export session data** → CSV.
3. Rename the file following the convention above and place it in the correct folder.

## Master dataset columns

### `master_long.parquet` / `master_long_clean.parquet` (1 Hz, granular)

| Column | Type | Description |
|--------|------|-------------|
| `User_ID` | str | Participant folder name (e.g. `P001_NomePart1`) |
| `Date` | date | Session date parsed from the filename |
| `Day_of_Week` | str | Full weekday name |
| `Time` | timedelta | Elapsed time within the session |
| `Time_s` | Int64 | Elapsed time in whole seconds |
| `HR_bpm` | float | Heart rate (bpm) — NaN where invalid or large gap |
| `Speed_kmh` | float | Speed (km/h) |
| `Pace_minkm` | float | Pace (min/km) |
| `Altitude_m` | float | Altitude (m) |
| `Distance_m` | float | Cumulative distance (m) |
| `Temp_C` | float | Ambient temperature (°C) |
| `gap_before` | bool | True when time gap from previous row > 2 s |
| `is_anomaly_zscore` | bool | \|z-score\| > 3 within session |
| `is_anomaly_iqr` | bool | Outside [Q1 − 1.5·IQR, Q3 + 1.5·IQR] |
| `is_anomaly` | bool | Union of both anomaly flags |

### `master_short.parquet` (one row per participant × day)

| Column | Description |
|--------|-------------|
| `User_ID`, `Date`, `Day_of_Week` | Identifiers |
| `FC_mean`, `FC_std`, `FC_variance` | HR central tendency and spread |
| `FC_min`, `FC_max`, `FC_range` | HR extremes |
| `duration_s` | Session length in seconds |
| `data_quality_pct` | % of 1 Hz rows with valid HR |
| `calories` | From session metadata |

## App móvel (BLE HRV collector)

A app React + Capacitor corre em Android e iOS e faz streaming BLE
do Polar H10 → RR intervals → Supabase.

### Pré-requisitos

| Ferramenta | Onde instalar |
|---|---|
| Node.js ≥ 20 | WSL (Ubuntu) |
| Android Studio | Windows |
| Chave Supabase | `app/.env` (ver `app/.env.example`) |

### Primeiro setup

```bash
cd app
npm install
cp .env.example .env          # preencher VITE_SUPABASE_URL e VITE_SUPABASE_ANON_KEY

# Adicionar plataforma Android (só uma vez)
npx cap add android
```

Depois de `npx cap add android`, adicionar permissões BLE ao
`android/app/src/main/AndroidManifest.xml` (ver secção abaixo).

### Desenvolvimento no browser

```bash
cd app && npm run dev           # abre em http://localhost:5173
```

O BLE no browser usa a Web Bluetooth API (Chrome/Edge) — clica
"Ligar à banda" para abrir o picker nativo.

### Build Android (WSL + Windows)

O Android Studio não consegue compilar a partir de caminhos `\\wsl$\...`.
O script `sync_to_windows.sh` trata de tudo:

```bash
cd app
./sync_to_windows.sh
```

O script faz automaticamente:
1. `npm run build` — compila a app web
2. `npx cap sync android` — sincroniza assets e plugins
3. Copia `android/` para `C:\Users\bruno\neroes_app_android\`

Depois, no **Android Studio (Windows)**:
- **File → Open** → `C:\Users\bruno\neroes_app_android`
- Aguarda o Gradle sync (primeira vez: 2–5 min)
- Liga telemóvel por USB com Depuração USB ativada
- Clica **▶ Run**

### Re-deploy após mudanças de código

```bash
cd app
./sync_to_windows.sh            # build + sync + copia para Windows
# No Android Studio: ▶ Run (usa a versão atualizada automaticamente)
```

### Gerar APK para distribuir

No Android Studio: **Build → Build Bundle(s) / APK(s) → Build APK(s)**

Ficheiro gerado em:
```
C:\Users\bruno\neroes_app_android\app\build\outputs\apk\debug\app-debug.apk
```

### Permissões BLE no AndroidManifest.xml

Adicionar dentro de `<manifest>`, antes de `<application>`:

```xml
<uses-permission android:name="android.permission.BLUETOOTH_SCAN"
    android:usesPermissionFlags="neverForLocation" tools:targetApi="s" />
<uses-permission android:name="android.permission.BLUETOOTH_CONNECT"
    tools:targetApi="s" />
<uses-permission android:name="android.permission.BLUETOOTH"
    android:maxSdkVersion="30" />
<uses-permission android:name="android.permission.BLUETOOTH_ADMIN"
    android:maxSdkVersion="30" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"
    android:maxSdkVersion="30" />
<uses-feature android:name="android.hardware.bluetooth_le" android:required="true" />
```

> O AndroidManifest.xml em `app/android/` já tem estas permissões.
> O `sync_to_windows.sh` copia-as para o Windows automaticamente.
